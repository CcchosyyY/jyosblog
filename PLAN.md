# 나만의 블로그 프로젝트 계획

> 여러 일들의 과정이나 내가 했던 일들을 정리할 수 있는 개인 블로그

---

## 1. 기술 스택

| 영역 | 기술 | 선택 이유 |
|------|------|-----------|
| **프론트엔드** | Next.js 14 (React) | SSG/SSR 지원, SEO 최적화, 빠른 페이지 로딩 |
| **스타일링** | Tailwind CSS | 빠른 UI 개발, 반응형 디자인 용이 |
| **콘텐츠** | MDX (Markdown + JSX) | 글 작성이 쉽고, 코드 블록/컴포넌트 삽입 가능 |
| **배포** | Vercel | Next.js 최적화, 무료 호스팅, 자동 배포 |
| **패키지 매니저** | npm | 의존성 관리 |

## 4. 프로젝트 폴더 구조

```
MyBlog/
├── app/                    # Next.js App Router
│   ├── layout.tsx          # 루트 레이아웃
│   ├── page.tsx            # 홈페이지
│   ├── blog/
│   │   ├── page.tsx        # 글 목록
│   │   └── [slug]/
│   │       └── page.tsx    # 글 상세
│   └── about/
│       └── page.tsx        # 소개 페이지
├── components/             # 재사용 컴포넌트
│   ├── Header.tsx
│   ├── Footer.tsx
│   ├── PostCard.tsx
│   └── ...
├── content/                # 블로그 글 (MDX)
│   └── posts/
│       ├── first-post.mdx
│       └── ...
├── lib/                    # 유틸리티 함수
├── styles/                 # 글로벌 스타일
├── public/                 # 정적 파일 (이미지 등)
├── tailwind.config.js
├── next.config.js
└── package.json
```

## 5. 진행 방식

1. **한 번에 하나씩**: 각 Step을 완료하고 확인한 후 다음으로 넘어가기
2. **자주 커밋**: 작은 단위로 git commit하여 진행 상황 기록
3. **중간중간 확인**: `npm run dev`로 결과물 확인하며 진행
4. **기록 남기기**: 막히는 부분이나 배운 것 메모해두기
